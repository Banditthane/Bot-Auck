## Objective

Plan a Discord slash-command moderation system for this repository. The initial requested capability is server-level moderation including ban and kick, with a coherent first-release command set.

## Expected behavior

Plan at minimum:

- `/ban` — ban a target member with an optional reason and a safe Discord-supported message-history deletion option if appropriate.
- `/kick` — remove a target member from the guild with an optional reason.
- `/unban` — remove a ban by Discord user ID, with an optional reason.
- `/timeout` — apply a bounded communication timeout with an optional reason.
- `/untimeout` — remove an active timeout with an optional reason.
- `/warn` — record or deliver a moderator warning only if the repository architecture can support clear, testable semantics without speculative infrastructure.

Planner may recommend a smaller Phase 1 when persistence, audit logging, or policy decisions would make `/warn` unsafe or underspecified. Clearly separate Phase 1 from later work.

## Important distinction

The existing `/room kick` command disconnects a member from a managed voice room. It is not guild moderation and must remain behaviorally unchanged. New moderation commands must not collide with or silently reuse the auto-voice-room service.
